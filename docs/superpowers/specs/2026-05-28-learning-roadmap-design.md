# 学习路线图可视化 — 设计文档

> 为 StudyAI 学习计划详情页增加交互式可视化路线图，用户可直观浏览学习路径、按天查看任务，并获取每阶段对应的学习资料推荐。

---

## 背景

当前学习计划详情页仅提供「时间线」（垂直阶段卡片 + 周/天任务列表）和「理论依据」两种视图。用户希望增加类似 airoadmaps.org 的节点式路线图，能够：

1. 一屏看清整个学习路径的全貌
2. 快速定位当前学习进度位置
3. 点击节点查看对应学习资料

---

## 架构决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 路线图位置 | 计划详情页顶部（概览卡片下方） | 不破坏现有时间线/理论依据 Tab，作为默认概览视图 |
| 布局方向 | 从左到右横向流程 | 直观展示时间推进的线性学习路径 |
| 节点粒度 | 默认显示阶段，点击展开周 | 初始简洁，渐进展开细节 |
| 任务展示 | 路线图下方按天分组 | 选中某周后，下方自动展示该周每天的任务卡片 |
| 资料面板 | 右侧固定面板，跟随选中天变化 | 点击某天即显示当天的论文/视频/代码 |
| 数据来源 | 基于现有 LearningPlan 数据结构 | 复用已有 stage → week → day → task 层级 |
| 学习资料 | AI 在生成计划时附带资料链接 | 与计划生成模型联动，非静态配置 |
| 渲染方案 | React 组件 + SVG/HTML 混合 | 节点用 HTML 卡片，连线用 SVG，保证样式和交互灵活性 |

## 组件结构

```
src/components/plan/
  learning-roadmap.tsx        ← 新建：路线图主组件（阶段节点 + 连线 + 展开逻辑）
  roadmap-stage-node.tsx      ← 新建：单个阶段节点卡片
```

## 数据模型扩展

```ts
// 新增类型
interface LearningResource {
  id: string
  title: string
  url: string
  type: "paper" | "video" | "code" | "article" | "book"
  source: string       // 如 "arxiv.org", "DeepLearning.AI", "GitHub"
  dayNumber: number    // 关联的天编号
  weekNumber: number   // 关联的周编号
}

// 扩展 DayPlan
interface DayPlan {
  // ... 现有字段
  resources?: LearningResource[]  // 当天的学习资料
}
```

## 交互行为

| 操作 | 效果 |
|------|------|
| 页面加载 | 路线图默认展示所有阶段节点，当前阶段高亮并自动展开周节点 |
| 点击阶段节点 | 展开/折叠该阶段的周节点列表 |
| 点击周节点 | 下方任务列表切换为该周按天分组视图，资料面板切换为该周概览资料 |
| 点击天卡片 | 该天卡片高亮选中（黄色边框 + 发光），右侧资料面板切换为该天的学习资料 |
| 勾选任务 | 任务打勾 + 进度条实时更新 + 数据持久化到 localStorage |
| 横向滚动 | 路线图区域支持横向滚动（touch + wheel），适配多阶段场景 |
| 移动端 | 路线图横向滚动，资料面板折叠到底部或隐藏（响应式） |
| Tab 切换 | 「时间线」「理论依据」Tab 保留在任务列表区域上方，可切换回原视图 |

## 视觉规范

- **阶段节点**：140-160px 宽圆角卡片，白色/灰色边框
  - 已完成：绿色边框 + 绿色序号 + ✓ 标记
  - 进行中：紫色边框 + 发光阴影 + 脉冲动画点
  - 未开始：灰色边框 + 灰色序号
- **周节点**：100-120px 宽小卡片，缩进在阶段下方
  - 已完成：灰色 + ✓
  - 当前/选中：黄色边框 + 星标
  - 未开始：浅灰
- **连线**：阶段之间 SVG 箭头线
  - 已完成→当前：紫色
  - 当前→未开始：灰色
  - 未开始→未开始：深灰
- **天卡片**：全宽圆角卡片，当前天黄色左边框高亮
- **资料面板**：230px 宽固定侧边栏，sticky 定位，分类显示论文/视频/代码

## 验证方式

1. 打开任意已创建的学习计划 → 页面顶部显示路线图概览
2. 点击阶段 2 → 展开周节点
3. 点击第 4 周 → 下方展示 Day 22-28 按天分组任务
4. 点击 Day 23 → 高亮 + 右侧资料面板显示对应资源
5. 勾选任务 → 进度实时更新
6. 切换到「时间线」Tab → 原有视图正常工作
7. 移动端测试 → 路线图可横向滑动，资料面板响应式折叠
8. `npm run build` 零错误
